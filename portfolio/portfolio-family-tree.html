Family Tree Application (Pending)

Family Tree is an idea to store, analyze and visualize family information more easily. It is quite similar to the Ancestry.com interface for families, family events, and individual information. The main difference is that it will rely on some base understanding of graph theory (not much more than people might understand from gossiping with friends) and reimagined visualizations.

Purpose

The purpose of this application is to make family information more readily accessible to everyone. Our ancestors have gone through so much to learn from their trials and to get to their envrinment that eventually led to us. They went through unimaginable events from an older world, through relevant troubles that we can learn from, and through problems that they eventually passed down to us, either culturally or genetically. History may not repeat itse;f, but it does rhyme. We can stand to benefit greatly if we can pay more attention to its prose.

We have much that we can learn from our past, so let's make it easier to access older family and friends information, as well as easier for the current generation to store their information for their progeny.

Technologies

This application will ultimate use the various technologies highlighted below:

Data Storage

Graph Database

The primary candidate database to host the relational data will be a Neo4j database. This way we can efficiently query sub-families and sub-networks from the larger family/friend network.

Relational Database

The goal of the database is to store time-variant attributes and events of individuals and relationships.

Document Database

This database will simply host the documents and will be referenced from the other databases.

Visualization

Simply because it's the fastest for me to whip out an interactive web application, I intend to use Dash Plotly.

Data Processing

Since this project is still very nascent, I plan to use python to remain flexible.
